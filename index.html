A brief overview of NX.js

NX.js is a powerful tool that provides a flexible and scalable approach to managing monorepo architecture.

  • React
  • Angular
  • Vue
  • Express/Nest/Fastify
  • Module Federation
  • Non-js, like Rust, Go, etc...

Challenges of managing complex monorepos

  • Code organization
  • Code reuse
  • Build optimization
  • Testing
  • Collaboration
  • Managing dependencies
  • Managing code generation
  • Managing code formatting
  • Managing code linting

The benefits of a monorepo architecture

  1. Code sharing
  2. Centralized management

Run Tasks

To build, test, lint, and format your code.

            nx run --target=build --projects=api

Cache Task Result

            "cacheableOperations": ["build", "test"]

Explore the Graph

                $ nx dep-graph

Distribute Task Execution (DTE)

Run tasks in parallel across multiple machines.

            $ nx run-many --target=build --projects=api --maxParallel=3

Automate Updating Dependencies

Automatically update dependencies across your workspace.

            $ nx migrate --run-migrations=migrations.json

Enforce Project Boundaries

Enforce project boundaries by defining which projects can import from which other projects.

            implicitDependencies": {
            "api": {
                "tags": ["your-tag", "your-other-tag"]
            }
        }

Tags are used to define groups of projects that can import from each other.

Customizable Builders and Executors

Customize the way your code is built, tested, and linted.

            "builders": {
            "build": {
                "builder": "@nrwl/node:build",
            }
        }
        
            "options": {
                "outputPath": "dist/apps/api",
                "main": "apps/api/src/main.ts",
                "tsConfig": "apps/api/tsconfig.app.json",
                "assets": [
                    "apps/api/src/assets"
                ]
            }
        
            "configurations": {
    "production": {
        "optimization": true,
        ...
        "fileReplacements": [
            {
                "replace": "environment.ts",
                "with": "environment.prod.ts"
            }
        ]
    }
            }
